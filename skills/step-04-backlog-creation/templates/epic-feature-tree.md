### 🤖 AGENT DIRECTIVE (HIDDEN FROM FINAL OUTPUT)
When the user requests an Epic/Feature Tree or Story Map, execute the following steps based on Jeff Patton's Story Mapping technique:
1. **The Backbone (Key Activities):** Identify the chronological flow of high-level activities a user must take to achieve the Product Goal (e.g., Register -> Find Product -> Checkout).
2. **The Walking Skeleton (Epics):** Break down each Key Activity into large Epics.
3. **The User Stories:** Decompose Epics into smaller, actionable user stories.
4. **Prioritization (Release Lines):** Vertically sort the stories into "Release 1.0 (Must-Have/MRFs)" and "Future Releases (Nice-to-Have)".
5. **Output Generation:** Output ONLY the "EPIC / FEATURE TREE" section below using markdown tables or nested lists.
